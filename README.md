# DeNotenman

Monorepo for the DeNotenman webshop and product importer.

Author: MNRV.

## Status

Bootstrap complete. Ready for sprint 1.

## First-time setup

```bash
corepack enable
pnpm install
cp .env.example .env
# Fill in .env (at minimum: SEED_OWNER_PASSWORD, SEED_ADMIN_PASSWORD, JWT secrets)

pnpm db:up                                       # postgres + redis + maildev
pnpm --filter @denotenman/prisma migrate:dev     # creates the initial migration
pnpm --filter @denotenman/prisma seed            # seeds categories, products, users, shipping rules
```

Verify:

```bash
pnpm --filter @denotenman/prisma studio          # opens Prisma Studio at localhost:5555
```

You should see 7 top-level categories, ~40 subcategories, 20 products with variants, 2 users, 2 shipping rules.

## Scripts (root)

| Command               | Does                                                 |
|-----------------------|------------------------------------------------------|
| `pnpm dev`            | Runs all apps in dev mode (turbo parallel)           |
| `pnpm build`          | Builds all apps                                      |
| `pnpm lint`           | ESLint across the workspace                          |
| `pnpm typecheck`      | `tsc --noEmit` across packages                       |
| `pnpm test`           | Vitest across packages                               |
| `pnpm test:e2e`       | Playwright end-to-end                                |
| `pnpm format`         | Prettier write                                       |
| `pnpm db:up`          | Docker compose up for postgres + redis + maildev     |
| `pnpm db:down`        | Docker compose down                                  |
| `pnpm db:reset`       | Reset DB (destructive, dev only)                     |
| `pnpm db:seed`        | Re-run the seed (idempotent)                         |

## Git hooks

Husky is installed by `pnpm install` (via the `prepare` script). Two hooks are active:

- **pre-commit**: runs `lint-staged` (Prettier + ESLint on staged files), then `scripts/hooks/no-attribution.sh` which blocks commits containing AI or tool attribution.
- **commit-msg**: runs `scripts/hooks/validate-commit-msg.sh` which enforces Conventional Commits and rejects attribution trailers.

Test the hooks locally:

```bash
# Should fail: AI reference in message
git commit --allow-empty -m "feat: generated by AI"

# Should fail: wrong format
git commit --allow-empty -m "Fixed the bug"

# Should pass
git commit --allow-empty -m "chore: bootstrap complete"
```

## Team & protocol

Seven sub-agents in `.claude/agents/` and seven skills in `.claude/skills/`. The orchestrator delegates, never writes code directly.

Consensus vote (architect + auditor + specialist, all APPROVE) required for schema changes, new runtime deps, auth/payments/webhook changes, architectural shifts, production deploys. Vote records live in `docs/votes/`. Architecture decisions in `docs/adr/`.

See `CLAUDE.md` for the full directive.

## Sprint 1 kickoff

Open the project in Antigravity, start the Claude Code session, and paste the kickoff command at the bottom of `denotenman-build-prompt.md`.

Expected sprint 1 deliverables:

- Monorepo scaffolded (`@app-admin` — you'll add `apps/storefront`, `apps/admin`, `apps/api`, `apps/worker` and `packages/*`).
- First Prisma migration generated and applied (`@database-dev`).
- `packages/schemas` Zod schemas for Product, Category, Cart (`@fullstack-dev`).
- API skeleton with `/v1/products` and `/v1/categories` reading from the DB (`@backend-dev`).
- Storefront home, category, and PDP pages wired to real data (`@frontend-dev`).
- Design tokens and base UI primitives in `packages/ui` (`@frontend-dev`).
- CI pipeline green on first PR (`@app-admin`).

Every merge passes through `@auditor`.
